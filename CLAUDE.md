# Qouta — Project Memory for Claude Code

## What is Qouta
A professional quotation and invoicing PWA (Progressive Web App) for small businesses. Live at https://qouta.site. Built with Next.js 14, Supabase, and deployed on Vercel via GitHub (repo: iTalkBS/quota).

## Developer Profile
- **Non-technical user** — no coding background
- Makes mistakes easily, especially with partial file edits
- Prefers complete file replacements over partial edits — ALWAYS give complete files
- Uses Windows PC with Command Prompt (cmd) — never PowerShell syntax
- Working from China — VPN causes network instability and slow Supabase responses
- Vibe coding approach — relies on Claude to catch all errors and edge cases
- Always confirm before building — present plan first, ask for approval

## Tech Stack
- **Frontend:** Next.js 14 (App Router), TypeScript, Tailwind CSS + custom CSS variables
- **Database:** Supabase (PostgreSQL + Auth with RLS)
- **Deployment:** Vercel + GitHub
- **PDF:** @react-pdf/renderer
- **PWA:** next-pwa
- **Domain:** qouta.site (Namecheap, configured on Vercel)

## Database Tables
- `profiles` — business settings per user
- `clients` — user clients with contact_person field
- `items_directory` — reusable products/services
- `quotes` — handles both quotes and invoices (type field: quote/invoice)
- `quote_items` — line items per quote/invoice
- `payments` — partial payment tracking per invoice

## Key Database Functions
- `get_next_quote_number(uid, doc_type)` — generates QUO-YYYY-XXXX or INV-YYYY-XXXX using sequences

## Important Rules
- ALWAYS give complete files — never partial edits. Partial edits cause corruption
- ALWAYS use `cmd` syntax not PowerShell
- NEVER use backtick template literals in files written via Node.js scripts — use string concatenation instead
- Cache keys: dashboard, documents, clients, profile (10 min TTL), default 60s TTL
- Clear relevant caches after any data mutation
- ActionBtn component must be defined OUTSIDE the main page component to avoid re-render issues
- Settings form fields must use useCallback to prevent typing focus loss

## Document Flow
1. Create quote → status: sent
2. Quote can be: accepted, rejected, converted, expired
3. Convert quote → invoice (due date auto-set 30 days, items copied explicitly)
4. Invoice can be: unpaid, partial, paid, overdue
5. Payments tracked in payments table, amount_paid synced to quotes table

## PDF Rules
- Generated server-side at /api/pdf route
- Ownership verified (user_id check) before generating
- Notes do NOT appear on PDF — internal use only
- Banking details hidden on paid invoices
- Footer: "Generated by Qouta · WhatsApp: +86 133 4225 2657"
- Contact person shows as "Attn: Name" under Bill To

## PDF Download
- Mobile (iPhone/Android): uses Web Share API (navigator.share)
- Desktop: uses blob download with createElement('a')
- Fallback: window.open if both fail

## Known Limitations
- iPhone PWA home screen icon only works when added from Safari. Chrome on iPhone ignores apple-touch-icon entirely — this is an Apple restriction, not fixable in code. Users must use Safari to add Qouta to their home screen.
- Settings reloads on every tab switch (Next.js component unmount/remount — architectural limitation)
- Supabase free tier has cold starts causing occasional slowness
